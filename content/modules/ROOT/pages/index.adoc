= LINBIT Edge Storage Workshop
:page-layout: home

== Overview

Welcome to the *LINBIT Edge Storage Workshop* -- a hands-on lab for deploying and operating high-performance persistent storage with *LINBIT SDS (LINSTOR + DRBD)* on *OpenShift 4.22+*.

This workshop targets Solutions Architects and platform engineers evaluating storage for edge, minimal-footprint, and two-node topologies where Ceph/ODF is impractical.

== What You Will Learn

By completing these modules you will be able to:

. Describe LINSTOR/DRBD architecture and explain the control-plane vs data-plane separation
. Configure topology-aware StorageClasses for database locality using `WaitForFirstConsumer`
. Perform zero-downtime VM live migration with block-level RWX and `allow-two-primaries`
. Demonstrate cluster resilience under node failure using DRBD quorum
. Execute incremental S3 snapshot shipping for geographic disaster recovery

== Cluster Topology

[tabs]
====
AWS (Default)::
+
Your cluster uses the *Two-Node with Arbiter (TNA)* or *Compact 3-node* topology on AWS EC2.
LINSTOR uses EBS-backed storage pools on primary nodes and a diskless tiebreaker on the arbiter node for quorum.

KVM / Bare Metal (Advanced)::
+
Your cluster uses the *Two-Node Fencing (TNF)* topology with Redfish STONITH.
LINSTOR uses local or virtio disks and two diskful replicas with Pacemaker-based fencing for split-brain prevention.
====
